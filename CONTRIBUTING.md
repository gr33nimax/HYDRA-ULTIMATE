# Contributing to VLESS Ultimate Installer

Спасибо за интерес к проекту! Ниже — правила участия.

## Как помочь

- **Баги** → открывайте Issue с шаблоном `bug_report.md`
- **Feature-запросы** → Issue с описанием задачи и use-case
- **Код** → Pull Request по процессу ниже
- **Документация** → правки в README, TROUBLESHOOTING, INSTALL приветствуются

## Процесс Pull Request

1. Сделайте fork репозитория
2. Создайте ветку от `master`: `git checkout -b fix/ingress-whitelist`
3. Внесите изменения — только в `_core.py` или shell-скрипты
4. Проверьте синтаксис: `python3 verify.py` должен пройти без ошибок
5. Опишите в PR: что сломано, как воспроизвести, что изменено

## Архитектура проекта

```
bootstrap.sh           — точка входа (curl | bash), проверка зависимостей
main.py                — тонкая обёртка: exec(_core.py)
vless_installer/
  _core.py             — ВСЯ логика (≈37k строк), единый файл намеренно
  __init__.py          — пустой, для импорта
  __all_exports.py     — реэкспорт публичных символов
verify.py              — pre-publish sanity check
```

`_core.py` намеренно монолитный — это упрощает `exec`-загрузку и
`curl | bash`-дистрибуцию без `pip install`. Не пытайтесь разбить его
на пакеты без предварительного обсуждения в Issue.

## Правила кода

- Python 3.10+, без внешних зависимостей кроме stdlib
- Все пользовательские сообщения — на русском (целевая аудитория)
- Функции именуются `_snake_case`, приватные — с префиксом `_`
- После каждого изменения правил iptables — проверяйте порядок цепочки:
  `ESTABLISHED,RELATED → ACCEPT` должен стоять **выше** любых `DROP`

## Что не принимается

- Изменения, ломающие `verify.py`
- Добавление внешних pip-зависимостей в основной код
- Хардкод IP-адресов или доменов (кроме публичных CDN для geo-данных)
- Логирование приватных ключей или токенов

## Тестирование

Минимальная проверка перед PR:

```bash
# Синтаксис Python
python3 -c "import ast; ast.parse(open('vless_installer/_core.py').read())"

# Полный verify
python3 verify.py

# Синтаксис bash
bash -n bootstrap.sh
```

Если у вас есть тестовый VPS — проверьте полный цикл установки.
Логи: `/var/log/vless-install.log`.

Отправляя PR, вы подтверждаете, что код написан вами и может быть
распространён под этой лицензией.

# Telegram Admin Bot

Бот — административный адаптер HYDRA. Доступ разрешён только Telegram user ID,
указанному как Admin Chat ID через TUI.

## Интерфейс

Команда `/start` или `/menu` открывает кнопочный Control Center:

- состояние VPS и сервисов;
- AntiDPI: чистая протокольная и сетевая телеметрия, запуск/остановка и разбан;
- Honeypot: порт ловушки, собственные баны, запуск/остановка и разбан;
- Fail2ban: сводка и подробности каждого jail — ошибки, баны и политика;
- настройки уведомлений.

Изменение состояния защитных сервисов и разбан требуют подтверждения. Команды
`/system`, `/antidpi`, `/honeypot`, `/fail2ban` и `/unban <ip>` доступны напрямую.

## Уведомления

Независимо переключаются master switch, AntiDPI, Honeypot, Fail2ban BAN,
события UNBAN и системные уведомления. Настройки атомарно сохраняются в state.
Выключенная категория не выполняет HTTP-запрос к Telegram. Honeypot уведомляет
только после подтверждённого успешного firewall-бана, а не при одном TCP connect.

Каждый AntiDPI ALERT содержит кнопку `🚫 Заблокировать`. Она применяет штатный
AntiDPI ipset-ban ко всему входящему трафику IP с прогрессивным TTL. Callback
принимается только от настроенного администратора, повторное нажатие не создаёт
новый offense, а адрес из whitelist заблокировать через кнопку нельзя. После
успеха исходный ALERT показывает `Action`, `TTL` и `Offense`, а кнопка удаляется.

## Безопасность

- IP валидируются и передаются внешним программам только как argv;
- динамический текст экранируется для Telegram HTML;
- токен не выводится при HTTP-ошибках;
- тяжёлые status/systemd операции выполняются вне asyncio event loop;
- изменение состояния сервисов и снятие блокировок требуют подтверждения;
- мониторы Fail2ban и Honeypot переживают logrotate и не воспроизводят старые события.

Бот работает от root. Токен является root-equivalent секретом: не добавляйте
бота в группы и перевыпустите токен при подозрении на компрометацию.

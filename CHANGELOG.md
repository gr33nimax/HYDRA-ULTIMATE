# Changelog

## [2.4.1] — 19 июля 2026

### Надёжность применения конфигурации

**Транзакционный оркестратор доведён до production-ready состояния.**

Теперь применение конфигурации:

- защищено потоковой и межпроцессной блокировкой;
- журналируется в `apply.jsonl`;
- проверяет состояние плагинов после запуска;
- сохраняет снимок nftables и конфигурации Sing-Box;
- автоматически откатывает частично применённые изменения при ошибке;
- сохраняет понятную причину последнего сбоя для CLI и диагностики.

### Состояние и миграции

**Хранилище `/var/lib/hydra/state.json` стало валидируемым и восстанавливаемым.**

- Добавлена структурная и семантическая проверка state.
- Повреждённый state восстанавливается из `.bak`, а исходный файл сохраняется как `.corrupt`.
- Запись выполняется атомарно через временный файл с `fsync`.
- State, резервная копия и lock-файл получают права `0600` на POSIX-системах.
- Сохранены миграции старых схем состояния.

### Пользователи

**Имя пользователя теперь может быть обычным идентификатором, а email остаётся поддерживаемым вариантом.**

- `testik` и `testik@example.com` принимаются одинаково корректно.
- Старые записи с email не мигрируются принудительно и продолжают работать.
- Конфигурации протоколов используют стабильный UUID пользователя.
- Секреты и credentials не попадают в безопасный CLI-статус.

### Headless CLI и диагностика

Добавлены JSON-команды для автоматизации:

```bash
sudo hydra status
sudo hydra validate
sudo hydra plan
sudo hydra apply --dry-run
sudo hydra apply
sudo hydra user list
```

Статус разделяет состояние конфигурации и фактическое состояние runtime-сервисов. В частности, DNSCrypt теперь определяется по systemd и по слушающему сокету, поэтому работающий сервис не отображается как выключенный.

### Sync Agent и Sing-Box

- Неудачные применения конфигурации ставятся в очередь на следующий цикл.
- Добавлена защита от параллельных запусков Sync Agent.
- Разделены автоматические проверки лимитов, WARP и обновлений.
- Полная ручная проверка игнорирует автоматические переключатели.
- Добавлено безопасное обновление Sing-Box Extended с preflight-проверкой, резервной копией и откатом.
- Исправлено сравнение составных версий Sing-Box.
- Кэш WARP и журнал Sync Agent вынесены в явные runtime-константы.

### Плагины и firewall

- Исправлен бесконечный цикл очистки Portscan-правила Fail2ban: очистка теперь ограничена.
- `configure()` AmneziaWG больше не изменяет state и не пишет конфигурацию на диск.
- WDTT использует общий механизм сохранения firewall-правил вместо собственной дублирующей реализации.
- Устранены обращения тестов к реальным `/etc`, `/var`, `/run` и `/sys`.
- Сохранена совместимость с существующими настройками протоколов.

### Безопасность и эксплуатация

- Усилена проверка загружаемых бинарников и ELF-файлов.
- Конфигурации и state записываются атомарно.
- Улучшена обработка ошибок systemd, downloader и firewall.
- Bootstrap подготавливает изолированное Python-окружение и команду `hydra`.

### CI и тестирование

- Добавлены lock-файлы зависимостей и единая конфигурация Ruff/pytest.
- CI проверяет Python 3.10–3.13 на pull request и быстрый smoke-набор на push в `dev`/`main`.
- Добавлен dependency audit.
- Полный локальный набор: **536 passed**.

### Что не менялось

- Существующие пользовательские state-файлы не удаляются и не переписываются без необходимости.
- Протоколы и их публичные форматы ссылок не удалялись в этой версии.
- Изменения направлены на применение, откат, диагностику, безопасность и тестируемость.

## [2.4.0] — 18 июля 2026

### Новые транспорты

Добавлены ShadowTLS v3 + Trojan, Hysteria2 с Salamander obfuscation и Snell v4; реализованы установка, настройка из TUI, управление пользователями и выдача клиентских конфигураций.

### Подписки и доступ

Расширена генерация конфигураций для NekoBox и Throne, сохранены цепочки ShadowTLS и режим TrustTunnel QUIC; блокировка и удаление пользователя теперь немедленно отзывают доступ.

### Мониторинг и маршрутизация

Переработаны экран трафика и просмотр сервисных логов, стабилизирован `traffic_daemon`, добавлен детальный мониторинг пользователей и повтор применения конфигурации после сбоя синхронизации. qWDTT интегрирован в общий контур маршрутизации.

### TUI и сеть

Унифицированы экраны управления протоколами и статусы, добавлены пресеты SNI/обфускации, повышена точность VPS-диагностики. Добавлен системный профиль автотюнинга прокси-нагрузки с резервной копией и откатом.

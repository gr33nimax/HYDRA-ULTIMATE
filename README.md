> **HYDRA** — это специализированный, легковесный менеджер подписок и конфигураций для обхода интернет-цензуры. Проект автоматически управляет учетными данными пользователей и синхронизирует настройки в современных stealth-протоколах: **NaiveProxy**, **AmneziaWG** и **Mieru**.

Разработано как форк оригинального проекта мультипротокольного инсталлятора, с упором на автоматизацию, неинтерактивное управление пользователями и предоставление отказоустойчивой системы подписок.

---

## 🛠️ Основной стек протоколов

* **NaiveProxy** — стойкий к активному пробингу HTTP/2-прокси с маскировкой под TLS-трафик браузера Chromium.
* **AmneziaWG** — модифицированный WireGuard с обфускацией заголовков и размеров пакетов для защиты от DPI-анализа.
* **Mieru** — mTLS-прокси со случайным заполнением пакетов (random padding) для борьбы со статистическим анализом трафика.

## 📖 Возможности платформы

### 1. Единая система подписок и авто-провижининг
* **Синхронизация учетных записей:** Добавление или удаление пользователя в системе подписок автоматически регистрирует или удаляет его credentials в активных инстансах **NaiveProxy**, **Mieru** и **AmneziaWG**.
* **Пакетная синхронизация (в 1 клик):** Автоматическая генерация и восстановление всех недостающих конфигураций для пользователей во всех установленных на сервере протоколах.
* **Гибкие форматы экспорта:**
  * **Base64** подписки для стандартных VPN-клиентов.
  * **Clash Meta (Mihomo) YAML** конфигурации.
  * **Sing-box JSON** конфигурации с авто-трансляцией протоколов во внешние outbounds.

### 2. Продвинутая интеграция с AmneziaWG (Docker)
* **Байтово-идентичная генерация ссылок:** Генератор ссылок `sn://awg?` формирует корректные данные для импорта в NekoBox / SagerNet, в точности следуя правилам упаковки varint-длин ключей и параметров обфускации (`Jc`, `Jmin`, `Jmax`, `S1-S4`, `H1-H4`, `I1`).
* **Консольные QR-коды:** Автоматический рендеринг QR-кодов для конфигураций непосредственно в терминале через `qrencode`.
* **Отказоустойчивое логирование:** Интеллектуальный fallback на чтение системного журнала `journalctl` в случае, если на хосте заблокирован стандартный вывод логов Docker.
* **Компактная статистика пиров:** Чистый табличный вывод информации об активных подключениях и потреблении трафика.

### 3. Telegram-уведомления
* Уведомления о критических системных событиях и изменении конфигураций, отправляемые администратору через Telegram-бота.

## 🔧 Структура меню управления подписками CLI

| Опция | Действие | Описание |
| :---: | :--- | :--- |
| **1** | Настроить сервис подписок | Привязка домена и автоматический выпуск SSL-сертификата (Certbot) |
| **2** | Включение / Выключение | Управление системной службой HTTP-сервера подписок |
| **3** | Добавить пользователя | Добавление в БД подписок и автоматическое создание учетных записей в Naive, Mieru, AWG |
| **4** | Удалить пользователя | Удаление из системы подписок и депровижининг во всех протоколах |
| **5** | Показать ссылки подписки | Отображение внешних URL подписок пользователя |
| **6** | Перегенерировать токен | Обновление токена доступа для безопасности подписки |
| **7** | Изменить порт подписок | Переназначение порта HTTP-сервера подписок |
| **8** | Лимиты трафика | Настройка ограничений потребления данных для пользователей |
| **9** | Временные пользователи (TTL) | Управление учетными записями с ограниченным временем жизни |
| **10** | Синхронизировать конфигурации | Автоматическая генерация отсутствующих конфигов для всех пользователей |

# HYDRA

Управление несколькими прокси-протоколами и сетевыми сервисами на одном Linux-сервере через единый терминальный интерфейс.

[![Version](https://img.shields.io/badge/version-2.3.5-2563eb)](https://github.com/gr33nimax/HYDRA-ULTIMATE)
[![Python](https://img.shields.io/badge/Python-3.10%2B-3776ab)](https://www.python.org/)
[![Platform](https://img.shields.io/badge/Ubuntu%20%7C%20Debian-amd64%20%7C%20arm64-64748b)](#требования)
[![License](https://img.shields.io/badge/license-GPL--3.0-16a34a)](LICENSE)

HYDRA устанавливает и настраивает транспортные протоколы, собирает их конфигурацию для Sing-Box Extended, управляет маршрутизацией и firewall, выдаёт клиентские конфигурации и показывает состояние сервисов в TUI.

> [!WARNING]
> Проект активно развивается и изменяет системные сетевые настройки. Используйте отдельный VPS или сделайте снимок системы перед установкой.

## Возможности

- единая TUI для установки, настройки и диагностики сервисов;
- транзакционное применение конфигурации с проверкой и откатом при ошибке;
- SNI-мультиплексирование TLS-транспортов через Caddy L4;
- маршрутизация через Sing-Box, nftables/TPROXY и Cloudflare WARP;
- пользователи, ссылки подключения и HTTPS-подписки с лимитами трафика;
- учёт трафика и просмотр активных соединений;
- защита сервера через Fail2ban, honeypot и списки блокировки;
- Telegram-бот, системная диагностика и журналирование.

## Поддерживаемые модули

| Категория | Модули |
|---|---|
| Транспорты | AmneziaWG 2.0, AnyTLS, TrustTunnel, Mieru, NaiveProxy, Telemt MTProxy, qWDTT |
| Маршрутизация и DNS | Cloudflare WARP, DNSCrypt-proxy |
| Безопасность | Fail2ban, Honeypot, IPBan |

Модули можно включать независимо. HYDRA соберёт общую конфигурацию, проверит её и применит только после успешной валидации.

## Требования

- чистый сервер с Ubuntu или Debian;
- архитектура `amd64` или `arm64`;
- Python 3.10 или новее;
- права `root`;
- публичный IP-адрес;
- домен, направленный на сервер, для транспортов с TLS-сертификатами.

Установщик сам добавляет базовые системные зависимости и Sing-Box Extended. Бинарные файлы отдельных модулей загружаются при их включении.

## Установка

```bash
curl -fsSL https://raw.githubusercontent.com/gr33nimax/HYDRA-ULTIMATE/main/bootstrap.sh | sudo bash
```

После установки панель доступна командой:

```bash
sudo hydra
```

Проект устанавливается в `/opt/hydra`. Состояние хранится в `/var/lib/hydra`, основные журналы — в `/var/log/hydra`.

### Установка из клона

```bash
git clone https://github.com/gr33nimax/HYDRA-ULTIMATE.git
cd HYDRA-ULTIMATE
sudo bash bootstrap.sh
```

## Как устроено применение настроек

```text
TUI
 └─ состояние HYDRA
     └─ плагины формируют фрагменты конфигурации
         └─ Sing-Box + Caddy L4 + nftables
             ├─ проверка конфигурации
             ├─ атомарная запись
             └─ перезапуск сервисов или откат
```

HYDRA сохраняет пользовательские настройки отдельно от сгенерированных конфигураций. Это позволяет повторно собрать сетевой стек после изменения протокола, маршрута или состава пользователей.

## Основные разделы TUI

- протоколы и сетевые модули;
- пользователи и клиентские конфигурации;
- подписки и лимиты трафика;
- активные соединения и статистика;
- безопасность и списки блокировки;
- диагностика, логи и управление сервисами.

## Обновление

Повторно запустите установщик:

```bash
curl -fsSL https://raw.githubusercontent.com/gr33nimax/HYDRA-ULTIMATE/main/bootstrap.sh | sudo bash
```

Перед обновлением рабочего сервера рекомендуется сохранить `/var/lib/hydra` и пользовательские конфигурации из `/etc`.

## Разработка

Для запуска тестов достаточно Python 3.10+ и `pytest`:

```bash
python -m pip install pytest
python -m pytest tests -q
```

Локальный запуск TUI на Linux:

```bash
sudo python3 main.py
```

Структура проекта:

```text
main.py                 точка входа
hydra/core/             состояние, оркестрация, Sing-Box, nftables, SNI
hydra/plugins/          транспортные, сетевые и защитные модули
hydra/services/         подписки, трафик, Telegram-бот и фоновые службы
hydra/ui/               TUI и диагностика
tests/                  автоматические тесты
bootstrap.sh            установщик и обновление
```

## Важные замечания

- HYDRA рассчитана на Linux-сервер и не поддерживает запуск сетевого стека в Windows или macOS.
- Не редактируйте сгенерированные конфигурации вручную: при следующем применении настроек они могут быть перезаписаны.
- Проверяйте правила вашего хостинг-провайдера и местное законодательство перед развёртыванием прокси-сервисов.

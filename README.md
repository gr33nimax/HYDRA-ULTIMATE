> **HYDRA** — автоматизированная платформа развёртывания и администрирования многопротокольных прокси-серверов на базе **Sing-Box** (используется как единый сетевой оркестратор). Модульная архитектура объединяет 14 плагинов (8 транспортов, 3 сетевые службы, 3 модуля безопасности) с централизованным роутингом, шифрованием DNS и политиками ограничения доступа.

> [!IMPORTANT]
> Платформа находится в стадии активного бета-тестирования. Рекомендуется развертывание на чистой операционной системе (Ubuntu/Debian).

## Архитектура трафика

```
                      Входящий TLS-трафик (Порт 443)
                                     │
                                     ▼
      ┌─────────────────────────────────────────────────────────────────────────────┐
      │                   Caddy L4 (SNI-мультиплексор / Mux)                        │
      └──────────────┬───────────────┬───────────────┬───────────────┬──────────────┘
                     │ [sni-1]       │ [sni-2]       │ [sni-3]       │ [unknown/default]
                     ▼               ▼               ▼               ▼
                NaiveProxy        AnyTLS        TrustTunnel     Сайт-заглушка (Decoy)
                 (:10443)        (:20444)        (:20445)       (Защита от сканирования)
                     │               │               │
                     └───────────────┼───────────────┘
                                     │
                                     ▼
  Клиент (AWG/Mieru/etc.) ───► nftables TPROXY
                                     │
                                     ▼
                              Sing-Box (inbound)
                                     │
                       ┌─────────────┴─────────────┐
                       │                           │
                       ▼ (Политики роутинга)        ▼
                WARP (outbound)            Direct (outbound)
                       │                           │
                       ▼                           ▼
                  Cloudflare                    Интернет
                       │
                 [DNSCrypt:5300] (Локальный шифрованный upstream)

  ──────────────────────────────────────────────────────────────────
  Модули безопасности (Security):
  ┌────────────────────────────────────────────────────────────────┐
  │  Fail2ban  ──► Анализ логов (sing-box/sshd) → Блокировка в nft │
  │  Honeypot  ──► Ловушка сканирования портов → Блокировка        │
  │  IPBan     ──► Статическое управление списками IP/CIDR/ASN     │
  └────────────────────────────────────────────────────────────────┘
```

Конвейер применения изменений в конфигурации:
`orchestrator` → `collect_fragments` (сбор фрагментов плагинов) → `generate_config` (генерация JSON) → `write_config` (запись на диск) → `nft.apply_tproxy` (конфигурирование nftables) → `reload` (перезапуск демонов Sing-Box и регенерация Caddy L4).

## Описание ключевых механизмов

### Cloudflare WARP & Выборочный роутинг

Плагин `warp` настраивает маршрутизацию определенных категорий трафика через Cloudflare. По умолчанию работает в режиме **split-tunneling**, перенаправляя через WARP только трафик, подходящий под включенные правила:

* **Пользовательские правила**: Интерактивное добавление доменов, IP и CIDR-подсетей через TUI.
* **Кастомные профили релеев**: Поддержка подключения сторонних WireGuard/AmneziaWG конфигураций релеев. Достаточно поместить любой `.conf` файл в директорию `/etc/hydra/warp_profiles/` (например, `Russia.conf`), и вы сможете гибко маршрутизировать любой из локальных или внешних списков правил через этот профиль.
* **Предустановленные списки правил (itdog.info)**:
  * **РФ-сервисы** (`outside-raw.lst`): Российские хосты (используется для защиты IP-адреса VPS от сканирования и детектирования зондами РКН).
  * **GEO-block** (`geoblock.lst`): Заблокированные в РФ зарубежные ресурсы.
  * **GoogleAI** (`google_ai.lst`): Домены ИИ-сервисов Google (Gemini, Google AI Studio и др.).
* **Синхронизация**: Демон `sync_agent` каждые 24 часа скачивает активные списки, обновляет конфигурацию Sing-Box и перезапускает сетевой стек без обрыва текущих сессий.
* **Интеграция с Sing-Box**: Начиная с Sing-Box 1.13.0+ интеграция производится нативно через структуру `endpoints` (без промежуточных tun/outbound интерфейсов). Поддерживаются все расширенные параметры AmneziaWG (включая CPS-подписи `I1`–`I5`, мусорные пакеты и заголовки `S1`–`S4`, `H1`–`H4`). Rules-генерация использует `"domain_suffix"` для автоматического проксирования всех поддоменов сайтов из списков.

### TLS SNI Мультиплексирование (Caddy L4)

При активации двух или более TLS-плагинов, а также при одиночном AnyTLS или TrustTunnel, HYDRA настраивает **Caddy L4** в качестве фронтенда:

* **SSL Passthrough**: Caddy L4 считывает поле SNI (Server Name Indication) из пакета Client Hello без расшифрования трафика и перенаправляет сессию на локальный порт нужного транспорта:
  * NaiveProxy → порт `10443`
  * AnyTLS → порт `20444`
  * TrustTunnel → порт `20445`
* **Активная маскировка (Decoy)**: Соединения без SNI или с некорректным доменным именем перенаправляются на локальный веб-сервер (сайт-заглушку) для предотвращения детектирования активным зондированием.
* **QUIC/UDP**: NaiveProxy и TrustTunnel могут использовать UDP/443 через raw UDP proxy Caddy. Одновременно активировать QUIC у обоих транспортов нельзя без QUIC-aware SNI-маршрутизации; HYDRA отклоняет такой конфликт до применения конфигурации. Для TrustTunnel доступны режимы `tcp`, `quic` и `both`, при этом существующие установки по умолчанию остаются в `tcp`.

## Диагностический модуль VPS

Интерактивный пульт VPS-диагностики (`Опция 8` в TUI) позволяет проводить неинтерактивное тестирование сетевых интерфейсов и аппаратных ресурсов:

| Тест | Инструмент | Выходные данные |
|---|---|---|
| Локация (GeoIP) | Опрос внешних баз | IP, ASN, страна, город для контроля смены региона. |
| Доступность ресурсов | Censorcheck | Контроль блокировок стримингов/ИИ (OpenAI, Netflix, YouTube). |
| Проверка DPI | Censorcheck DPI | Анализ блокировок и деградации трафика на ТСПУ РФ. |
| Скорость до РФ | iPerf3 | Пропускная способность (Download/Upload) и RTT до точек обмена в РФ. |
| Global Speedtest | HTTP-загрузка файлов | Скорость скачивания и Ping до 14 мировых дата-центров. |
| Производительность CPU | Sysbench | Оценка вычислительной мощности процессора в один поток. |

## Установка и запуск

### Системные требования
* **ОС**: Ubuntu 20.04+, Debian 11+ (чистая установка)
* **Аппаратные ресурсы**: RAM от 512 МБ, ROM от 2 ГБ, внешний IPv4
* **Интерпретатор**: Python >= 3.10
* **Права**: суперпользователь (`root`)

### Быстрая установка (Рекомендуется)
```bash
bash <(curl -fsSL https://raw.githubusercontent.com/gr33nimax/HYDRA-ULTIMATE/main/bootstrap.sh)
```
*Скрипт автоматически обновит пакетную базу, установит Sing-Box Extended, склонирует файлы проекта в `/opt/hydra` и создаст symlink `/usr/local/bin/hydra`.*

### Ручная установка
```bash
git clone https://github.com/gr33nimax/HYDRA-ULTIMATE /opt/hydra
cd /opt/hydra
sudo python3 main.py
```

### Первоначальная настройка
1. Откройте консольное меню: `sudo hydra`.
2. Включите необходимые транспорты в разделе **Протоколы**.
3. Активируйте системные службы (**DNSCrypt**, **WARP**).
4. Настройте правила безопасности (**Fail2ban**, **Honeypot**).
5. Создайте пользователей в разделе **Пользователи** для экспорта конфигурационных ссылок и QR-кодов.

Проверка состояния сервисов:
```bash
sudo hydra status
journalctl -u sing-box -n 20 --no-pager
```

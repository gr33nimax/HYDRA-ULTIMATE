> **HYDRA** — это современная, легковесная платформа для автоматического развертывания безопасных прокси-серверов, управления учетными данными пользователей и предоставления отказоустойчивой системы подписок. Проект ориентирован на обход DPI-блокировок и интернет-цензуры с использованием современного stealth-стека: **AmneziaWG**, **NaiveProxy** и **Mieru** с интеграцией **system-wide DNSCrypt** и **WARP**.

---

## 🛠️ Современный стек протоколов

* **AmneziaWG** (Docker) — модифицированный WireGuard с обфускацией пакетов (размеров и заголовков) для обхода DPI-анализа.
* **NaiveProxy** — прокси, маскирующийся под стандартный TLS-трафик браузера Chromium (использует веб-сервер Caddy), устойчивый к активному зондированию (active probing).
* **Mieru** — mTLS-прокси с шумовым заполнением пакетов (random padding) для борьбы со статистическим анализом.
* **WARP (Cloudflare)** — система туннелирования внешнего трафика для изменения исходящего IP-адреса и обхода региональных ограничений.

## 📖 Возможности и функционал платформы

### 1. Централизованный жизненный цикл пользователей (`user_lifecycle.py`)
* **Автоматический провижининг**: Добавление, удаление, блокировка или разблокировка пользователя мгновенно синхронизирует его учетные данные во всех активных службах: AmneziaWG (Docker-контейнер), NaiveProxy (Caddyfile) и Mieru.
* **Автоматическая миграция**: Бот плавно мигрирует устаревших пользователей из структуры `sub_tokens` в единую базу данных `users` в `state.json` «на лету» без нарушения работы существующих клиентов.

### 2. Умный подсчёт трафика и защита от сброса счётчиков
* **Сводный подсчёт**: Собирает данные по потреблению трафика со всех служб (логи запросов NaiveProxy, системный журнал Mieru, сетевые счетчики интерфейсов AmneziaWG через Docker-вызовы).
* **Защита от ротации логов**: Учитывает ротацию файлов логов, перезапуск Docker-контейнеров или сброс сетевых интерфейсов, сохраняя накопленное потребление (`traffic_accumulated`) и вычисляя точное смещение (`traffic_baseline`).
* **Парсер единиц WireGuard**: Автоматически преобразует человекочитаемые объемы данных WireGuard (KiB, MiB, GiB, TiB) в точные байты для предотвращения потери статистики.

### 3. Фоновый Sync Agent на Systemd
* Работает через встроенный системный таймер и однократную службу (`hydra-sync-agent.timer` / `hydra-sync-agent.service`) каждые 5 минут.
* Проверяет ограничения по лимитам трафика (GB) и сроку действия подписок (TTL) для каждого пользователя.
* Автоматически блокирует превысивших лимит клиентов (удаляет из конфигов активных VPN, сохраняя учетную запись в БД как заблокированную) и отправляет уведомление администратору в Telegram.
* Автоматически разблокирует пользователей при продлении подписки или увеличении лимита трафика.
* Полностью заменяет и исключает устаревшие cron-задачи.

### 4. Системный DNSCrypt-Proxy (Шифрование DNS)
* **Прозрачное проксирование**: Служба DNSCrypt-proxy (`127.0.0.1:5300`) интегрирована на уровне ОС через `systemd-resolved` (или `/etc/resolv.conf`).
* **iptables NAT REDIRECT**: Любые DNS-запросы (порт 53 UDP/TCP) от VPN-клиентов AmneziaWG перехватываются на уровне брандмауэра и прозрачно шифруются через DoH/DNSCrypt.
* **RTT latency-тест**: Интерактивный CLI-тест скорости пинга до безопасных резолверов автоматически подбирает наиболее быстрые DNS-серверы для текущей локации вашего сервера.

### 5. Универсальная маршрутизация через WARP
* **Маршрутизация на уровне ядра**: Позволяет заворачивать трафик к определенным доменам (например, `openai.com`, `claude.ai`) и IP-подсетям через сетевой интерфейс Cloudflare WARP на уровне операционной системы.
* Применяется ко всем прокси-службам (NaiveProxy, Mieru, AmneziaWG) одновременно.

### 6. Двухботовая система Telegram (Admin Bot & Client Bot)
Бот запускается в фоновом режиме через две независимые systemd-службы (`xray-tg-admin.service` и `xray-tg-bot.service`) с использованием легковесных Python-оберток.
* **Панель Администратора**:
  * **Управление пользователями (`/users`)**: Интерактивное меню со списком клиентов, постраничной пагинацией и inline-кнопками для детального просмотра, блокировки, разблокировки, удаления или изменения TTL/лимита.
  * **Прямые команды**: Быстрое создание подписок через `/addsub <email>` и удаление через `/delsub <email>`.
  * **Интерактивные диалоги**: Изменение лимитов данных и TTL через текстовые ответы во временных состояниях.
  * **Панель безопасности (`/security`)**: Агрегатор системных логов, статуса блокировок Fail2Ban и активности Honeypot.
  * **Метрики трафика (`/traffic`)**: Сводное потребление ресурсов в реальном времени с разделением по протоколам.
* **Кабинет Пользователя**:
  * **Управление подпиской (`/config` / `/start`)**: Выдача персональных ссылок для мобильных клиентов (Base64) и ПК (Throne/NekoBox).
  * **Статистика в реальном времени**: Вывод текущего потребления трафика, графической полосы прогресса (progress bar) лимита и даты окончания действия подписки.
  * **QR-коды (`/qr`)**: Генерация консольных и графических QR-кодов для быстрого импорта подписок на мобильные устройства.

Для развертывания или обновления HYDRA выполните команду в консоли чистого сервера под управлением Ubuntu/Debian:
```bash
sudo bash -c "$(curl -fsSL https://raw.githubusercontent.com/gr33nimax/HYDRA-ULTIMATE/main/bootstrap.sh)"
```

## 🔧 Структура меню управления CLI (`main.py`)

Запустите интерактивную консоль управления с помощью команды `python3 main.py`:

* **Основное меню**:
  * Управление пользователями подписок (Добавление, удаление, просмотр токенов).
  * Настройка системных лимитов данных и сроков действия доступа (TTL).
  * Быстрая синхронизация конфигураций пользователей во всех протоколах.
  * Управление сетевыми параметрами и универсальной маршрутизацией WARP.
  * Тонкая настройка системного DNSCrypt-proxy и тестирование RTT.
  * Настройка интеграции Telegram-ботов и управление системными службами.

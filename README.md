> **HYDRA** — менеджер многопротокольного прокси-сервера на базе **Sing-Box Extended**. Проект объединяет 12 плагинов: 7 транспортов, 2 сетевые службы и 3 модуля безопасности. Конфигурация, маршрутизация, пользователи, подписки и диагностика управляются из единого TUI.

> [!IMPORTANT]
> Платформа находится в стадии активного бета-тестирования. Рекомендуется развертывание на чистой операционной системе (Ubuntu/Debian).

## Архитектура трафика

```text
TCP/443 ──► Caddy L4 (маршрутизация по SNI)
              ├─► NaiveProxy/Caddy ──► SOCKS inbound Sing-Box
              ├─► AnyTLS inbound Sing-Box
              ├─► TrustTunnel TCP inbound Sing-Box
              └─► decoy для неизвестного SNI

UDP/443 ──► Caddy L4 raw UDP proxy ──► NaiveProxy QUIC
                                      или TrustTunnel QUIC

AmneziaWG ──► nftables TPROXY ──┐
Mieru ──────────────────────────┼─► Sing-Box ──► правила маршрутизации
TLS-транспорты ─────────────────┘                   ├─► direct
                                                   ├─► Cloudflare WARP
                                                   └─► WG/AWG relay profile

DNSCrypt-proxy :5300 ──► локальный шифрованный DNS upstream
Telemt и qWDTT ────────► отдельные systemd-сервисы

Fail2ban ──► анализ journald/access-log ──► блокировка
Honeypot ──► обнаружение сканирования ────► блокировка
IPBan ─────► ручные правила IP/CIDR/ASN ──► firewall
```

Конвейер применения изменений:
`snapshot state` → `collect_fragments` → `generate_config` → проверка и атомарная запись JSON → применение nftables и плагинов → reload Sing-Box → перестроение Caddy L4. При ошибке изменение состояния и предыдущая конфигурация Sing-Box восстанавливаются.

## Протоколы и службы

### Транспорты (TRANSPORT)

| Протокол | Системное имя | Реализация | Статус |
|---|---|---|---|
| AmneziaWG 2.0 | `amneziawg` | Kernel-модуль Linux; трафик передаётся в Sing-Box через TPROXY | Готов |
| Mieru | `mieru` | Нативный inbound Sing-Box, mTLS и настраиваемый random padding | Готов |
| NaiveProxy | `naive` | Caddy forward proxy; режимы HTTP/2, QUIC или оба транспорта | Готов |
| MTProto | `telemt` | Telemt MTProxy с несколькими пользователями и отдельными secret | Готов |
| qWDTT | `wdtt` | WireGuard поверх VK TURN/DTLS с парольной авторизацией | Готов |
| AnyTLS | `anytls` | Нативный inbound Sing-Box с padding scheme | Готов |
| TrustTunnel | `trusttunnel` | Нативные HTTP/2 и QUIC inbound; режимы TCP, QUIC или оба | Готов |

### Сетевые службы (ENHANCEMENT)

| Служба | Описание | Статус |
|---|---|---|
| DNSCrypt | Системный DNSCrypt-proxy; локальный шифрованный DNS upstream на порту 5300 | Готов |
| WARP | Выборочная маршрутизация через Cloudflare WARP или пользовательские WG/AWG-профили | Готов |

Telegram-бот не входит в реестр плагинов: это отдельная служба управления пользователями и подписками.

### Безопасность (SECURITY)

| Плагин | Назначение | Статус |
|---|---|---|
| Fail2ban | Изолированные jail для SSH и протоколов, recidive и опциональная защита от portscan | Готов |
| Honeypot | TCP-ловушка с IPv4/IPv6-блокировкой обнаруженных сканеров | Готов |
| IPBan | Ручная блокировка IPv4/IPv6, CIDR, диапазонов и ASN | Готов |

## Ключевые механизмы

### Cloudflare WARP & Выборочный роутинг

Плагин `warp` настраивает маршрутизацию определенных категорий трафика через Cloudflare. По умолчанию работает в режиме **split-tunneling**, перенаправляя через WARP только трафик, подходящий под включенные правила:

* **Пользовательские правила**: Интерактивное добавление доменов, IP и CIDR-подсетей через TUI.
* **Кастомные профили релеев**: Поддержка подключения сторонних WireGuard/AmneziaWG конфигураций релеев. Достаточно поместить любой `.conf` файл в директорию `/etc/hydra/warp_profiles/` (например, `Russia.conf`), и вы сможете гибко маршрутизировать любой из локальных или внешних списков правил через этот профиль.
* **Предустановленные списки правил (itdog.info)**:
  * **РФ-сервисы** (`outside-raw.lst`): Российские хосты (используется для защиты IP-адреса VPS от сканирования и детектирования зондами РКН).
  * **GEO-block** (`geoblock.lst`): Заблокированные в РФ зарубежные ресурсы.
  * **GoogleAI** (`google_ai.lst`): Домены ИИ-сервисов Google (Gemini, Google AI Studio и др.).
* **Синхронизация**: Демон `sync_agent` каждые 24 часа скачивает активные списки, обновляет конфигурацию Sing-Box и применяет сетевую конфигурацию.
* **Интеграция с Sing-Box**: Начиная с Sing-Box 1.13.0+ интеграция производится нативно через структуру `endpoints` (без промежуточных tun/outbound интерфейсов). Поддерживаются все расширенные параметры AmneziaWG (включая CPS-подписи `I1`–`I5`, мусорные пакеты и заголовки `S1`–`S4`, `H1`–`H4`). Rules-генерация использует `"domain_suffix"` для автоматического проксирования всех поддоменов сайтов из списков.

### TLS SNI Мультиплексирование (Caddy L4)

При активации двух или более TLS-плагинов, а также при одиночном AnyTLS или TrustTunnel, HYDRA настраивает **Caddy L4** в качестве фронтенда:

* **SSL Passthrough**: Caddy L4 считывает поле SNI (Server Name Indication) из пакета Client Hello без расшифрования трафика и перенаправляет сессию на локальный порт нужного транспорта:
  * NaiveProxy → порт `10443`
  * AnyTLS → порт `20444`
  * TrustTunnel → порт `20445`
* **Активная маскировка (Decoy)**: Соединения без SNI или с некорректным доменным именем перенаправляются на локальный веб-сервер (сайт-заглушку) для предотвращения детектирования активным зондированием.
* **QUIC/UDP**: NaiveProxy и TrustTunnel могут использовать UDP/443 через raw UDP proxy Caddy. UDP-маршрут не разбирает SNI, поэтому владельцем внешнего UDP/443 может быть только один из этих транспортов. Конфликт проверяется до применения конфигурации.

## Диагностика VPS

Интерактивный пульт VPS-диагностики (`Опция 8` в TUI) позволяет проводить неинтерактивное тестирование сетевых интерфейсов и аппаратных ресурсов:

| Тест | Инструмент | Выходные данные |
|---|---|---|
| Локация (GeoIP) | Опрос внешних баз | IP, ASN, страна, город для контроля смены региона. |
| Доступность ресурсов | Censorcheck | Контроль блокировок стримингов/ИИ (OpenAI, Netflix, YouTube). |
| Проверка DPI | Censorcheck DPI | Анализ блокировок и деградации трафика на ТСПУ РФ. |
| Скорость до РФ | iPerf3 | Пропускная способность (Download/Upload) и RTT до точек обмена в РФ. |
| Global Speedtest | HTTP-загрузка файлов | Скорость скачивания и Ping до 14 мировых дата-центров. |
| Производительность CPU | Sysbench | Оценка вычислительной мощности процессора в один поток. |

## Установка и запуск

### Системные требования
* **ОС**: Ubuntu или Debian с `systemd` (рекомендуется чистая установка)
* **Архитектура**: `amd64` или `arm64`
* **Интерпретатор**: Python >= 3.10
* **Права**: суперпользователь (`root`)
* **Сеть**: публичный IP; домен для транспортов, использующих TLS-сертификаты

### Быстрая установка
```bash
curl -fsSL https://raw.githubusercontent.com/gr33nimax/HYDRA-ULTIMATE/main/bootstrap.sh | sudo bash
```
Скрипт проверит ОС и Python, установит системные зависимости и Sing-Box Extended, развернёт проект из ветки `main` в `/opt/hydra` и создаст ссылку `/usr/local/bin/hydra`.

### Установка из клона
```bash
git clone https://github.com/gr33nimax/HYDRA-ULTIMATE.git
cd HYDRA-ULTIMATE
sudo bash bootstrap.sh
```

### Первоначальная настройка
1. Откройте консольное меню: `sudo hydra`.
2. Включите необходимые транспорты в разделе **Протоколы**.
3. Активируйте системные службы (**DNSCrypt**, **WARP**).
4. Настройте правила безопасности (**Fail2ban**, **Honeypot**).
5. Создайте пользователей в разделе **Пользователи** для экспорта конфигурационных ссылок и QR-кодов.

Основной журнал HYDRA и журнал Sing-Box:
```bash
sudo tail -n 100 /var/log/hydra/install.log
journalctl -u sing-box -n 20 --no-pager
```

### Файлы и каталоги

| Путь | Назначение |
|---|---|
| `/opt/hydra` | исходный код установленного экземпляра |
| `/var/lib/hydra/state.json` | состояние HYDRA; записывается атомарно, рядом создаётся резервная копия |
| `/etc/sing-box/config.json` | сгенерированная конфигурация Sing-Box |
| `/etc/hydra/warp_profiles/` | пользовательские профили WireGuard/AmneziaWG для маршрутизации |
| `/var/log/hydra/` | журналы, диагностический отчёт и отладочные файлы |

> [!WARNING]
> Не редактируйте сгенерированные конфигурации вручную: следующее применение настроек перезапишет их. Перед обновлением рабочего сервера сохраните `/var/lib/hydra` и пользовательские файлы из `/etc`.

## История изменений (Changelog)

### [2.3.5] — 2026-07-17

#### Исправлено
* Унифицирован транзакционный lifecycle плагинов: `configure → validate → apply → commit/rollback`.
* Добавлена межпроцессная блокировка и атомарное обновление общего состояния.
* Исправлены удаление правил UFW/iptables и безопасная работа с общими портами.
* nftables и конфигурация Caddy теперь проверяются до замены рабочей конфигурации.

* Учёт трафика сделан монотонным: сброс интерфейса и ротация access-log больше не уменьшают использованную квоту.
* Активные подключения показываются по атрибутированным сессиям Clash API; внутренние proxy-соединения и неактивные AWG-пиры скрыты.
* Единицы трафика приведены к IEC-обозначениям KiB/MiB/GiB.
* Добавлена безопасная распаковка загружаемых tar-архивов.

## Разработка и тестирование

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/gr33nimax/HYDRA-ULTIMATE.git
   cd HYDRA-ULTIMATE
   ```
2. Установите зависимости тестирования:
   ```bash
   python -m pip install pytest
   ```
3. Запустите тесты:
   ```bash
   python -m pytest tests -q
   ```
4. Локальный запуск TUI (требуются права `root` и ОС семейства Linux):
   ```bash
   sudo python3 main.py
   ```

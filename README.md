# 🐉 HYDRA (v0.0.1-alpha)

[![Version](https://img.shields.io/badge/version-0.0.1--alpha-red.svg)]()
[![Python](https://img.shields.io/badge/python-3.10%2B-green.svg)]()
[![Platform Support](https://img.shields.io/badge/platform-Ubuntu%20%7C%20Debian-lightgrey.svg)]()

**HYDRA** — это легковесный менеджер подписок и конфигураций для обхода интернет-цензуры на базе современных stealth-протоколов: **NaiveProxy**, **AmneziaWG** и **Mieru**.

Проект ориентирован на автоматическое и неинтерактивное управление учетными записями пользователей, генерацию кроссплатформенных подписок и предоставление удобного CLI-интерфейса для администраторов.

---

## 🚀 Основной стек протоколов

* **NaiveProxy** — стойкий к активному пробингу прокси с маскировкой под стандартный TLS-трафик веб-браузера Chromium.
* **AmneziaWG** — обфусцированная версия WireGuard, устойчивая к DPI-анализу и блокировкам стандартных VPN-протоколов.
* **Mieru** — mTLS прокси с добавлением случайного заполнения (random padding), работающий без привязки к домену.

---

## 📖 Ключевые возможности

### 📋 Управление подписками
* **Автоматическое связывание:** Добавление или удаление пользователя в системе подписок мгновенно регистрирует или удаляет его учетные данные в активных инстансах **NaiveProxy**, **Mieru** и **AmneziaWG**.
* **Синхронизация в один клик:** Функция пакетного обновления проверяет базу данных подписок и автоматически генерирует недостающие конфигурации для всех зарегистрированных пользователей в установленных на сервере протоколах.
* **Генерация кроссплатформенных ссылок:**
  * **Base64** подписки для v2rayNG, Hiddify, Shadowrocket.
  * **Clash Meta (Mihomo) YAML** конфигурации.
  * **Sing-box JSON** конфигурации (с корректной трансляцией NaiveProxy и Mieru во внешние outbounds).
  * **NekoBox/SFA `sn://awg` ссылки** — генерация байтово-идентичных ссылок для AmneziaWG с использованием правильного varint-префикса длин ключей и параметров обфускации.

### 🛡️ Управление AmneziaWG (Docker)
* **Интерактивный показ клиентских конфигов:** Удобный выбор пользователя с выводом текстового `.conf` файла, сгенерированной `sn://awg` ссылки и QR-кода прямо в консоль (через `qrencode`).
* **Таблица активных соединений:** Компактный вывод статуса пиров с отображением потребляемого трафика, времени последней активности и IP-адресов.
* **Резервное чтение логов:** Автоматический переход на чтение системного журнала `journalctl` в случаях, когда Docker настроен на сторонние драйверы логирования, не поддерживающие прямое чтение логов контейнера.

## 🔧 CLI Интерфейс управления подписками

В меню подписок доступны следующие действия:
1. **Настроить сервис подписок (Домен + SSL)**
2. **Включение / Выключение службы подписок**
3. **Добавить пользователя подписок** (автоматически провижнит Naive, Mieru, AWG)
4. **Удалить пользователя подписок** (автоматически удаляет credentials)
5. **Показать ссылки подписки пользователя**
6. **Перегенерировать токен доступа**
7. **Изменение порта HTTP-сервера подписок**
8. **Управление лимитами трафика**
9. **Временные пользователи (TTL)**
10. **Синхронизировать и обновить все конфигурации**

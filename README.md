**HYDRA** — это автоматизированная модульная платформа развёртывания, оркестрации и администрирования многопротокольных прокси-серверов на базе **Sing-Box** (используется как единое сетевое ядро). 

Платформа объединяет 14 плагинов (транспорты, сетевые службы, модули безопасности) с централизованной маршрутизацией, DNS-шифрованием, автоматическим распределением портов по SNI с помощью Caddy L4, гибким биллингом и интерактивным TUI-интерфейсом управления.

> [!IMPORTANT]
> Платформа находится в стадии активного бета-тестирования. Рекомендуется развертывание на чистой операционной системе (Ubuntu 20.04+ / Debian 11+). Для работы скриптов требуются права суперпользователя (`root`).

## 📐 Архитектура системы

HYDRA использует двухуровневую обработку трафика: входящие TLS-соединения мультиплексируются по SNI с помощью **Caddy L4**, а прочие протоколы (например, UDP/TPROXY) обрабатываются непосредственно в ядре **Sing-Box**.

### Схема прохождения трафика

```
                      Входящий TLS-трафик (Порт 443)
                                     │
                                     ▼
      ┌─────────────────────────────────────────────────────────────────────────────┐
      │                   Caddy L4 (SNI-мультиплексор / Mux)                        │
      └──────────────┬───────────────┬───────────────┬───────────────┬──────────────┘
                     │ [sni-1]       │ [sni-2]       │ [sni-3]       │ [unknown/default]
                     ▼               ▼               ▼               ▼
                NaiveProxy        AnyTLS        TrustTunnel     Сайт-заглушка (Decoy)
                 (:10443)        (:20444)        (:20445)       (Защита от сканирования)
                     │               │               │                   │
                     └───────────────┼───────────────┘                   ▼
                                     │                            Локальные папки
                                     ▼                           /var/www/decoy-*/
  Клиенты (AWG/Mieru/etc.) ──► nftables TPROXY
                                     │
                                     ▼
                             Sing-Box (inbound)
                                     │
                       ┌─────────────┴─────────────┐
                       │                           │
                       ▼ (Политики роутинга)        ▼
                WARP (outbound)            Direct (outbound)
                       │                           │
                       ▼                           ▼
                  Cloudflare                    Интернет
                       │
                 [DNSCrypt:5300] (Локальный шифрованный upstream)

  ──────────────────────────────────────────────────────────────────
  Модули безопасности (Security):
  ┌────────────────────────────────────────────────────────────────┐
  │  Fail2ban  ──► Анализ логов (sing-box/sshd) → Блокировка в nft │
  │  Honeypot  ──► Ловушка сканирования портов → Блокировка        │
  │  IPBan     ──► Статическое управление списками IP/CIDR/ASN     │
  └────────────────────────────────────────────────────────────────┘
```


### Конвейер применения конфигурации (Transaction Lifecycle)

Все изменения настроек в TUI применяются атомарно по транзакционному принципу. В случае ошибки на любом этапе происходит автоматический откат к предыдущему стабильному состоянию системы и базы данных.

```
  TUI (main.py) ────► Запись в AppState (state.json)
                            │
                            ▼ Применение (orchestrator.py)
  1. Сбор фрагментов ───────► registry.collect_fragments()
  2. Генерация конфига ─────► singbox.generate_config()
  3. Валидация изменений ───► Тест структуры JSON Sing-Box & правил nftables
                            │
            ┌───────────────┴───────────────┐
            ▼ Успешно                       ▼ Сбой
  Запись конфигурации на диск        Откат состояния AppState
  Перезапуск служб через Systemd     Восстановление рабочей конфигурации
```

## 🔑 Ключевые механизмы

### 1. Cloudflare WARP & Выборочный роутинг
По умолчанию работает в режиме **split-tunneling**, перенаправляя через WARP только трафик, подходящий под включенные правила:
* **Кастомные профили релеев**: Поддержка подключения сторонних WireGuard/AmneziaWG конфигураций релеев. Достаточно поместить `.conf` файл в `/etc/hydra/warp_profiles/` (например, `Russia.conf`), и вы сможете гибко маршрутизировать любой из списков правил через этот профиль.
* **Предустановленные списки правил (itdog.info)**:
  * **РФ-сервисы** (`outside-raw.lst`): Российские хосты (используется для защиты IP-адреса VPS от сканирования и детектирования зондами РКН).
  * **GEO-block** (`geoblock.lst`): Заблокированные в РФ зарубежные ресурсы.
  * **GoogleAI** (`google_ai.lst`): Домены ИИ-сервисов Google (Gemini, Google AI Studio и др.).
* **Синхронизация**: Демон `sync_agent` каждые 24 часа скачивает активные списки, обновляет конфигурацию Sing-Box и перезапускает сетевой стек без обрыва текущих сессий.

### 2. TLS SNI Мультиплексирование (Caddy L4)
При активации двух или более TLS-плагинов, а также при одиночном AnyTLS или TrustTunnel, HYDRA настраивает **Caddy L4** в качестве фронтенда:
* **SSL Passthrough**: Caddy L4 считывает поле SNI (Server Name Indication) из пакета Client Hello без расшифрования трафика и перенаправляет сессию на локальный порт нужного транспорта.
* **Активная маскировка (Decoy)**: Соединения без SNI или с некорректным доменным именем перенаправляются на локальный веб-сервер (сайт-заглушку в директории `/var/www/decoy-*/`) для предотвращения детектирования активным зондированием.
* **QUIC/UDP**: Поддержка работы NaiveProxy и TrustTunnel по UDP/443 через raw UDP proxy Caddy (с автоматическим отклонением конфликтов при одновременной активации).

### 3. Биллинг & HTTP Сервер подписок
* Сервер подписок работает на порту `9443` по протоколу HTTPS.
* Поддерживает формат NekoBox (`sn://`) для AmneziaWG, Mieru, Naive и др.
* **Учет трафика (traffic_daemon)**: Фоновый демон собирает статистику потребления трафика из Clash API Sing-Box, агрегирует данные и блокирует пользователей при исчерпании лимитов. Учет трафика является монотонным и устойчивым к ротации логов.
* **Моментальный отзыв**: При блокировке или удалении пользователя его сессии и токены на сервере подписок аннулируются мгновенно.

### 4. Диагностический модуль VPS
Интерактивный пульт VPS-диагностики (Опция 8 в TUI) позволяет проводить тестирование сетевых интерфейсов и аппаратных ресурсов:
* **GeoIP локация**: Опрос внешних баз для контроля смены региона IP, ASN, страны.
* **Censorcheck**: Контроль блокировок стримингов/ИИ (OpenAI, Netflix, YouTube).
* **Censorcheck DPI**: Анализ блокировок и деградации трафика на ТСПУ РФ.
* **RTT / Скорость до РФ**: Измерение пропускной способности iPerf3 до точек обмена в РФ.
* **Global Speedtest**: Скорость скачивания и Ping до 14 мировых дата-центров.
* **Производительность CPU**: Вычислительный тест Sysbench в один поток.

## 💿 Установка и запуск

### Системные требования
* **ОС**: Ubuntu 20.04+, Debian 11+ (рекомендуется чистая установка)
* **Ресурсы**: ОЗУ от 512 МБ, ПЗУ от 2 ГБ, внешний белый IPv4-адрес
* **Интерпретатор**: Python >= 3.10
* **Права**: `root`

### Быстрая установка (Рекомендуется)
Для автоматической установки сетевого стека, Caddy L4, Sing-Box Extended и создания символических ссылок выполните:
```bash
bash <(curl -fsSL https://raw.githubusercontent.com/gr33nimax/HYDRA-ULTIMATE/main/bootstrap.sh)
```
*Скрипт автоматически установит необходимые зависимости, настроит systemd-службы и скомпилирует caddy-l4 с нужными модулями.*

### Ручная установка
Если вы хотите развернуть проект вручную из исходного кода:
```bash
# Клонирование репозитория в директорию по умолчанию
git clone https://github.com/gr33nimax/HYDRA-ULTIMATE /opt/hydra
cd /opt/hydra

# Запуск панели управления (требуется Python 3.10+)
sudo python3 main.py
```

### Первоначальная настройка
1. Запустите консольное меню: `sudo hydra` (или `sudo python3 main.py`).
2. Перейдите в раздел **Протоколы** и включите необходимые вам транспорты.
3. Активируйте системные службы (**DNSCrypt**, **WARP**) для защиты DNS-запросов и обхода блокировок.
4. Перейдите в **Пользователи** и создайте аккаунты. Платформа сгенерирует ссылки и QR-коды для подключения.
5. Проверьте статус служб в TUI или с помощью системных команд:
```bash
sudo hydra status
journalctl -u sing-box -n 20 --no-pager
```

## 🧪 Разработка и тестирование

Проект покрыт развитой системой юнит-тестов (более 230 тестов), проверяющих генерацию конфигураций, транзакционный жизненный цикл плагинов, работу Caddy L4 и биллинг-демона.

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/gr33nimax/HYDRA-ULTIMATE.git
   cd HYDRA-ULTIMATE
   ```
2. Установите pytest:
   ```bash
   pip install pytest
   ```
3. Запустите тесты:
   ```bash
   python -m pytest tests/ -v
   ```
   *Все тесты выполняются изолированно в течение ~7 секунд.*

4. Для локальной отладки TUI (требуется Linux c правами `root`):
   ```bash
   sudo python3 main.py
   ```
